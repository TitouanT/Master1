Exercice 1

Base de faits et base de règles

Soit le programme suivant :

(deffacts faits-initiaux
    (statut pompe active)
    (statut vanne ouverte)
    (alerte pompe non)
    (statut alarme active))

(defrule regle1
    (declare (salience 5))
    (statut ?element active)
    =>
    (printout t "Je déclenche la règle 1" crlf))

(defrule regle2
    (statut ?element ouverte)
    (alerte ?element non)
    =>
    (printout t "Je déclenche la règle 2" crlf))
    
(defrule regle3
    (declare (salience 10))
    (statut ?element ouverte)
    (alerte ? non)
    =>
    (printout t "Je déclenche la règle 3" crlf))

(defrule regle4
    (statut $?etat)
    =>
    (printout t "Je déclenche la règle 4" crlf))
  • Quel sera le contenu de l'agenda après un (reset)?
  • Quel sera l'affichage lors de l'exécution du programme ?

(deffacts faits-initiaux
    (phrase le chat mange la souris))

(defrule regle-1
    (phrase $?debut ? $?reste)

    (assert (combinaison ?debut ?reste)))

Exercice 9

Manipuler des références (fact-address)

  • Modifiez le programme de l'exercice 5 de façon à ce que chaque fait créé conserve une référence vers le fait précédent (sorte de liste chaînée à l'envers) ;
  • Écrivez une fonction récursive qui remonte la liste à partir d'une référence passée en argument en affichant les entiers ;
  • Ajoutez une règle qui fait appel à cette fonction dès que le nombre 100 est atteint.

Vous utiliserez les fonctions suivantes :

  • Pour définir une fonction : (deffunction <nom> (<args>) <corps>)
  • Pour effectuer un test : (if <condition> then <action> [else <action>])
  • Pour afficher une donnée : (printout t <expr> crlf)
  • Pour accéder aux données d'un fait ordonné : (fact-slot-value <fact-address> implied)
  • Pour accéder aux nième élément d'une liste : (nth$ <position> <liste>)
  • Pour affecter une variable avec une valeur : (bind <variable> <expr>)

Attention : Ne mémorisez pas un fact-address si le fait est susceptible d'être modifié. En effet, toute modification d'un fait provoque également la modification de son fact-address. Il faut alors référencer le fait à l'aide d'un identificateur, comme ci-dessous :

(deftemplate fact
    (slot id (default-dynamic (gensym)))
    (slot ... ))
  • Modifiez votre programme en utilisant cette technique.

Exercice 10

Encore une fonction récursive

Soit le programme suivant :

(deffacts fait-initiaux
    (mot "coucou")
    (mot "laval")
    (mot "palindrome"))

(defrule palindrome
    (mot ?mot)
    =>
    (if (palindrome ?mot)
        then (printout t ?mot " est un palindrome" crlf)
        else (printout t ?mot " n'est pas un palindrome" crlf)))
  • Écrivez la fonction booléenne récursive palindrome en utilisant les fonctions suivantes :
    • Pour obtenir la longueur d'une chaîne de caractères : (str-length <chaine>)
    • Pour extraire une sous-chaîne : (sub-string <début> <fin> <chaine>)

Exercice 11

Utiliser une fonction d'évaluation

Un problème classique en IA est d'identifier un fait parmi plusieurs, en maximisant (ou en minimisant) une fonction d'évaluation (stratégie, diagnostic, optimisation, etc.). Pour illustrer ce principe, vous allez faire 3 petits exercices :

  • Vous allez commencer par identifier parmi les faits (nombre ...) celui qui contient la valeur la plus grande :
(deffacts fait-initiaux
    (nombre 35)
    (nombre 15)
    (nombre 10)
    (nombre 52)
    (nombre 23)
    (nombre 19))

(defrule rechercher-max
    ...
    =>
    (printout t "Le nombre le plus grand est " ?max crlf))
  • Une autre méthode consiste à supprimer les faits les moins intéressants afin de ne conserver que celui (ou ceux) qui sont les plus "prometteurs" :
(defrule supprimer-min
    ...
    =>
    (retract ?fait))
  • Pour terminer, vous allez écrire une fonction d'évaluation dédiée qui retourne un entier (élémentaire ici, puisqu'il s'agit de l'entier contenu dans le fait), puis l'appliquer dans une règle qui applique la méthode n°2 (supprimer les faits les moins intéressants).
